Kako postaviti stranicu besplatno na GitHub Pages

1. Napravi GitHub račun

Ako nemaš već, otvori račun na GitHub.

2. Kreiraj novi repozitorij

  1. Klikni na New Repository.
  2. Nazovi ga, npr. asseria-gradnja.
  3. Odaberi opciju "Public" i klikni Create repository.

3. Dodaj svoje fajlove

  1. Na računaru napravi folder i u njega spremi index.html, sve slike i ostale fajlove.
  2. U GitHub repozitoriju klikni Upload files i ubaci sve fajlove iz tog foldera.
  3. Commitaj promjene.

4. Aktiviraj GitHub Pages

  1. Idi u Settings repozitorija.
  2. Nađi sekciju Pages (u meniju sa strane).
  3. U "Source" odaberi Main branch i klikni Save.
  4. GitHub će ti generirati link oblika: https://tvoje-ime.github.io/asseria-gradnja/

5. Dodaj vlastitu domenu

  1. Kupi domenu (npr. asseria-gradnja.hr) kod bilo kojeg registra domena (Plus.hr, GoDaddy, Namecheap...).
  2. U tvom GitHub repozitoriju idi na Settings > Pages.
  3. U polje Custom domain upiši svoju domenu (npr. asseria-gradnja.hr) i spremi.
  4. U kontrolnoj ploči gdje si kupio domenu, pronađi DNS postavke.
  5. Dodaj A zapise koji upućuju na IP adrese GitHub Pages:
    • 185.199.108.153
    • 185.199.109.153
    • 185.199.110.153
    • 185.199.111.153
  6. Dodaj i CNAME zapis koji pokazuje na tvoje-korisnicko-ime.github.io.

Primjer DNS zapisa

; A zapisi za GitHub Pages
@   3600 IN A     185.199.108.153
@   3600 IN A     185.199.109.153
@   3600 IN A     185.199.110.153
@   3600 IN A     185.199.111.153

; CNAME zapis
www 3600 IN CNAME tvoje-korisnicko-ime.github.io.

6. Provjeri povezivanje

Nakon što DNS promjene propagiraju (može potrajati do 24 sata), tvoja stranica će biti dostupna na tvojoj vlastitoj domeni.

Savjet

Kad god izmijeniš index.html ili druge fajlove, commitaj ih na GitHub i stranica će se automatski ažurirati i na tvojoj domeni.

Vodič za Plus.hr DNS postavke

Evo primjer kako to izgleda unutar Plus.hr kontrolne ploče:

  1. Prijavi se na Plus.hr korisnički panel.
  2. Otvori sekciju DNS Zone Editor.
  3. Klikni Dodaj zapis i unesi svaki od A zapisa zasebno.
  4. Dodaj i CNAME zapis za www.
  5. Spremi promjene.
